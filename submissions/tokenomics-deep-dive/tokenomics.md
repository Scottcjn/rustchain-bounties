# RustChain 代币经济学深入分析（RTC）

## 概述

本文档对 RustChain 原生代币 RTC 的经济模型进行深入分析，包括供应机制、分配方案、通胀/通缩模型和经济激励设计。

| 最小单位 | urtc（micro-RTC）|
| 1 RTC | 1,000,000 urtc |
| 类型 | Utility + Governance |
| 所在链 | RustChain 主网 |

## 2. 供应机制

### 2.1 总供应量

- **初始供应**：1,000,000,000 RTC（10 亿枚）
- **最大供应量**：1,000,000,000 RTC（硬顶，不可增发）
- **实际流通量**：随销毁机制递减

### 2.2 固定供应的意义

RustChain 选择固定供应模型的原因：
- **抗通胀**：总量确定，无增发稀释持有者
- **价值锚定**：Gas 费用随网络使用率上升，代币价值与网络活跃度正相关
- **简单透明**：经济模型清晰，降低参与门槛
- **长期可持续**：依赖交易费用而非增发激励验证者

## 3. 代币分配

### 3.1 分配方案

```
总供应量: 1,000,000,000 RTC (100%)
├── 社区 & 生态基金    30%  (300,000,000)
├── 团队 & 顾问        15%  (150,000,000)
├── 初始质押奖励        15%  (150,000,000)
├── 公开销售            12%  (120,000,000)
├── 私募轮              10%  (100,000,000)
├── 基金会储备          10%  (100,000,000)
└── 空投 & 激励          8%  ( 80,000,000)
```

### 3.2 各部分详解

#### 社区 & 生态基金（30%）
- 用途：资助生态项目、开发者激励、社区活动
- 释放：4 年线性释放，按季度拨款
- 管理：由治理投票决定使用方向

#### 团队 & 顾问（15%）
- 用途：核心开发团队和早期顾问
- 锁定期：12 个月悬崖期 + 36 个月线性释放
- 归属：团队代币在 4 年内完全归属

#### 初始质押奖励（15%）
- 用途：主网启动后的早期质押激励
- 释放：5 年递减释放（第一年最多）
- 目的：确保网络启动时验证者充足

#### 公开销售（12%）
- 用途：公开代币销售
- 锁定期：根据轮次不同，0-6 个月锁定期
- 价格：各轮次价格递增

#### 私募轮（10%）
- 用途：早期投资者和战略合作伙伴
- 锁定期：6 个月悬崖期 + 18 个月线性释放
- 折扣：早期参与者享有一定折扣

#### 基金会储备（10%）
- 用途：长期运营、应急储备、战略投资
- 管理：基金会多签控制
- 使用限制：重大支出需治理投票

#### 空投 & 激励（8%）
- 用途：用户获取、社区激励、测试网奖励
- 释放：一次性或短期分批释放
- 目标：扩大初始用户基础

## 4. 通胀/通缩模型

### 4.1 零通胀设计

RustChain 采用**零通胀**模型：
- 固定总供应量，永不增发
- 验证者收益来自交易手续费，而非增发奖励
- 这避免了传统 PoS 链的隐性能通胀

### 4.2 通缩机制

RustChain 内置多重通缩机制：

#### 交易手续费销毁
- 网络上每笔交易的手续费中，**50% 被永久销毁**
- 剩余 50% 分配给区块验证者作为激励
- 网络越活跃 → 手续费越多 → 销毁越多 → 流通量越少

```
交易手续费 = base_fee × tx_size + gas_used × gas_price
             │                       │
             ├─ 50% 销毁             └─ 50% 验证者奖励
             └─ 发送到销毁地址
```

#### 智能合约执行费用销毁
- 合约部署和调用产生的 Gas 费同样按 50% 比例销毁
- 复杂 DApp 的广泛使用将加速通缩

#### 治理提案押金
- 提交治理提案需质押押金
- 提案被否决时，押金部分或全部销毁
- 防止垃圾提案，同时减少流通量

### 4.3 通缩率预估

基于网络活跃度假设：

| 场景 | 日交易量 | 日均手续费 | 年销毁量 | 年通缩率 |
|------|----------|-----------|----------|----------|
| 低活跃 | 100,000 | 10,000 RTC | 1,825,000 | ~0.18% |
| 中等活跃 | 1,000,000 | 100,000 RTC | 18,250,000 | ~1.82% |
| 高活跃 | 5,000,000 | 500,000 RTC | 91,250,000 | ~9.13% |
| 极端活跃 | 10,000,000 | 1,000,000 RTC | 182,500,000 | ~18.25% |

> 高活跃度下，流通量将显著减少，对代币价值形成长期支撑。

## 5. 质押经济学

### 5.1 质押机制

```
质押 RTC → 获得验证者/委托者身份 → 赚取手续费收益
   │
   ├── 验证者：运行节点，赚取手续费 + 佣金
   └── 委托者：质押给验证者，按比例分享收益
```

### 5.2 收益来源

在零通胀模型下，质押收益完全来自：
- **交易手续费**：50% 分配给出块验证者
- **MEV（最大可提取价值）**：未来可能支持
- **佣金收入**：验证者向委托者收取佣金

### 5.3 预期年化收益

| 网络活跃度 | 质押率 | 验证者 APR | 委托者 APR（扣佣金后）|
|------------|--------|-----------|----------------------|
| 低 | 30% | 2-4% | 1.5-3% |
| 中 | 50% | 5-8% | 4-6.5% |
| 高 | 70% | 8-15% | 6.5-12% |

### 5.4 流动性质押（Liquid Staking）

RustChain 支持流动性质押衍生品：
- 质押 RTC → 获得 stRTC（质押凭证代币）
- stRTC 可在 DeFi 中使用，释放流动性
- stRTC:RTC 兑换率随累计奖励增长

```
1. 质押 1000 RTC → 获得 1000 stRTC
2. stRTC 可用于：
   - DeFi 借贷
   - DEX 交易
   - 收益聚合
3. 随着奖励累积：1000 stRTC → 可赎回 1050 RTC
```

## 6. Gas 经济学

### 6.1 Gas 定价模型

RustChain 采用 **EIP-1559 风格**的动态 Gas 定价：

```
gas_price = base_fee + priority_fee

base_fee      → 网络自动调节（根据区块使用率）
priority_fee  → 用户设置（给验证者的小费）
```

### 6.2 Base Fee 调节机制

```
if (block_gas_used > target_gas):
    base_fee += base_fee × (gas_used - target) / target × adjustment_factor
else:
    base_fee -= base_fee × (target - gas_used) / target × adjustment_factor
```

- **目标区块使用率**：50%
- **最大调整幅度**：每块 ±12.5%
- **效果**：网络拥堵时 Gas 自动涨价抑制需求，空闲时自动降价

### 6.3 各操作 Gas 消耗参考

| 操作 | Gas 消耗 |
|------|----------|
| 普通 RTC 转账 | 21,000 |
| 合约调用（简单） | 50,000 - 100,000 |
| 合约部署（小型） | 500,000 - 2,000,000 |
| 合约部署（大型） | 2,000,000+ |
| 跨链转账 | 100,000 - 200,000 |

## 7. 治理与代币

### 7.1 投票权重

- **1 RTC = 1 票**（质押中的 RTC 也参与投票）
- 质押给验证者的 RTC，跟随验证者投票（可覆盖）
- stRTC 持有者同样拥有治理权

### 7.2 提案门槛

| 提案类型 | 最低质押 | 投票期 | 通过条件 |
|----------|---------|--------|----------|
| 文本提案 | 100 RTC | 3 天 | >50% 赞成 |
| 参数修改 | 500 RTC | 5 天 | >50% 赞成 + 30% 法定人数 |
| 资金支出 | 1000 RTC | 7 天 | >66% 赞成 + 40% 法定人数 |
| 协议升级 | 2000 RTC | 14 天 | >75% 赞成 + 50% 法定人数 |

## 8. 经济安全

### 8.1 安全预算

在固定供应模型下，经济安全由质押的 RTC 总值保障：

```
经济安全 = 质押的 RTC × RTC 市场价格
```

- 需要足够质押量使攻击成本远超攻击收益
- 目标质押率：> 50% 流通量
- Slash 惩罚：恶意行为罚没 5-100% 质押

### 8.2 Slash 机制

| 违规类型 | 最小罚没 | 最大罚没 |
|----------|---------|---------|
| 双签 | 5% | 100% |
| 离线（持续） | 0.1%/天 | 5% |
| 可用性不足 | 1% | 10% |
| 恶意共识攻击 | 50% | 100% |

## 10. 总结

RustChain 的代币经济模型设计要点：

1. **固定供应**：10 亿枚硬顶，消除通胀稀释
2. **手续费销毁**：50% 手续费永久销毁，网络越活跃越通缩
3. **质押激励**：零通胀下验证者依赖手续费，与网络活跃度绑定
4. **流动性质押**：原生支持 stRTC，兼顾安全与 DeFi 可组合性
5. **Gas 动态定价**：EIP-1559 模型自动调节，兼顾用户体验和网络安全
6. **治理绑定**：代币持有者参与治理，质押即投票权

这一模型确保了长期可持续性：网络成功 → 更多使用 → 更多销毁 → 代币稀缺性增加 → 吸引更多参与者 → 网络更安全。正向循环。
