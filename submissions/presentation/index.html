What is RustChain?

A lightweight blockchain ecosystem written in Rust that rewards vintage hardware over modern silicon.

  • Anti-arms-race by design
  • Ultra-low overhead — runs on < 32 MB RAM
  • Hardware diversity as a security model
  • Rust for memory safety & bare-metal performance

The Problem with Modern Blockchain

  • 🔥 E-waste skyrocketing — functional hardware in landfills
  • 🏭 Centralization — mining controlled by industrial farms
  • 💰 Priced out — 32 ETH (~$80,000) to validate on Ethereum
  • ⚡ Energy waste — ASICs consuming megawatts for trust

The original promise of blockchain — anyone can participate — is dead.

Consensus: Proof of Antiquity (PoA)

Rewards the age and authenticity of physical hardware.

  • Hardware fingerprinting — 6+1 cryptographic checks
  • Anti-emulation — VMs and containers rejected (0x)
  • Vintage multiplier — older, rarer silicon earns more
  • Physical commitment — SHA-256(nonce + wallet + entropy)

Hardware Fingerprinting: 6+1 Checks

Multiplier Formula

Time Decay (Vintage Hardware)

decay_factor = 1.0 - (0.15 × years_since_genesis)
final_multiplier = 1.0 + (vintage_bonus × decay_factor)

Vintage bonus decays 15% per year from chain genesis (2025).

Loyalty Bonus (Modern Hardware)

+15% per year uptime, capped at +50% (1.5x total)

Server Hardware Bonus

Xeon / EPYC / Opteron → flat +10%

Security Model

Multi-layered Defense

  1. Nonce replay protection — 10-min TTL
  2. 6+1 fingerprint checks — VM/emulator detection
  3. No mock signatures — mandatory admin key
  4. Validated JSON — strict schema
  5. Hardware serial verification — manufacturing DB cross-check
  6. Antiquity multiplier — disincentivizes farms
Chain security: no_mock_sigs | mandatory_admin_key | replay_protection | validated_json
