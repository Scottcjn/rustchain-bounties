# Galaga 街机硬件规格 (1981)

## 📊 概述

Galaga 是由 Namco 于 1981 年 9 月发布的固定射击街机游戏。它是 Galaxian (1979) 的续作，使用 Namco Galaga 专用街机基板。

---

## 🔧 技术规格

### 中央处理器 (CPU)

| 属性 | 规格 |
|------|------|
| **主 CPU** | Zilog Z80 |
| **时钟频率** | 3.072 MHz |
| **架构** | 8 位 |
| **寻址空间** | 64 KB |
| **指令集** | 178 条基础指令 |
| **寄存器** | 8 个 8 位通用寄存器 (A, B, C, D, E, H, L, F) |
| **变址寄存器** | IX, IY (16 位) |
| **程序计数器** | PC (16 位) |
| **堆栈指针** | SP (16 位) |

### Z80 CPU 特性

```
Z80 寄存器组:
+-----+-----+-----+-----+-----+-----+-----+-----+
|  A  |  F  |  B  |  C  |  D  |  E  |  H  |  L  |  (主组)
+-----+-----+-----+-----+-----+-----+-----+-----+
| A'  | F'  | B'  | C'  | D'  | E'  | H'  | L'  |  (备用组)
+-----+-----+-----+-----+-----+-----+-----+-----+
|  I  |  R  |  I  |  X  |  I  |  Y  |  S  |  P  |  (特殊)
+-----+-----+-----+-----+-----+-----+-----+-----+

时钟周期:
- 基础指令: 4-8 T-states
- 内存访问: +3 T-states
- 刷新周期: 每 64 T-states
```

### 内存配置

| 类型 | 容量 | 用途 |
|------|------|------|
| **主 RAM** | 6 KB (6144 字节) | 游戏状态、精灵位置、分数 |
| **视频 RAM** | 2 KB (2048 字节) | 帧缓冲、调色板 |
| **ROM** | 24 KB | 游戏代码、图形数据、音频 |
| **总计可用** | 8 KB RAM | 极其受限 |

## 🎮 Galaga 游戏特性

### 游戏机制

- **固定射击**: 玩家飞船只能在屏幕底部左右移动
- **编队攻击**: 外星人以"蜜蜂编队"模式移动
- **捕获机制**: 某些外星人可以捕获玩家飞船
- **双飞船**: 营救被捕获的飞船获得双倍火力

### 敌人类型

| 敌人 | 行为 | 分数 |
|------|------|------|
| **Galaga** | 基础敌人，编队飞行 | 150-300 |
| **Goei** | 快速俯冲攻击 | 200-400 |
| **Boss** | 发射捕获光束 | 500-800 |
| **Butterfly** | 快速不规则移动 | 300-600 |
| **Tanks** | 慢速，高耐久 | 400-700 |

### 关卡设计

- **关卡 1-3**: 基础编队
- **关卡 4+**: 增加 Boss 和捕获机制
- **奖励关卡**: 每 2 关一次 (Challenging Stage)
- **难度递增**: 速度加快，模式复杂化

## 🔬 硬件指纹特征

### Z80 时序特征

Z80 CPU 具有独特的时序特征，可用于硬件指纹识别：

1. **时钟偏移**: 模拟振荡器随温度漂移
2. **指令时序**: 不同指令的 T-state 变化
3. **内存等待状态**: 访问不同内存区域的时间差异
4. **刷新周期**: DRAM 刷新引入的微小延迟

```python
# Z80 时序指纹示例
def z80_timing_fingerprint():
    """捕获 Z80 时序特征"""
    samples = []
    
    # 测量不同指令的执行时间
    for i in range(100):
        start = perf_counter()
        
        # NOP (4 T-states)
        for _ in range(1000):
            pass
        
        elapsed = perf_counter() - start
        samples.append(elapsed)
    
    # 统计分析
    mean_time = sum(samples) / len(samples)
    variance = sum((x - mean_time)**2 for x in samples) / len(samples)
    
    return {
        'mean': mean_time,
        'variance': variance,
        'samples': samples
    }
```

### 视频时序特征

CRT 显示器具有独特的模拟特征：

1. **水平扫描频率**: 15734.264 Hz ± 0.01%
2. **垂直扫描频率**: 59.94 Hz ± 0.01%
3. **色彩相位**: RGB 信号相位关系
4. **扫描线**: 224 条可见扫描线

### PSG 音频特征

音频芯片具有独特的频率响应：

1. **通道频率**: 每个通道的精确频率
2. **波形谐波**: 谐波失真特征
3. **包络时序**: ADSR 包络的精确时序

## 🔧 移植考虑

### 限制因素

1. **RAM 极度受限**: 6KB 无法容纳现代 TLS 栈
2. **无网络硬件**: 原始设计无网络功能
3. **8 位架构**: 无法直接处理 32/64 位加密运算
4. **ROM 固化**: 游戏逻辑硬编码，无法修改

### 可行方案

1. **Badge 实现**: 使用 ESP32 模拟 Galaga 硬件
2. **协同处理**: 外部模块处理网络，Z80 仅负责指纹
3. **离线模式**: 生成本地证明，稍后提交
4. **模拟器移植**: 在 MAME 等模拟器中运行矿工

### 推荐方法

**Badge-Only 解决方案** (如 Tron Handheld bounty #1875):

- 使用 ESP32-S3 模拟 Z80 行为
- 保持 Galaga 美学和控制布局
- IPS 显示屏模拟 CRT 效果
- WiFi 处理网络通信
- 外部尺寸与原始 Galaga 控制台一致
