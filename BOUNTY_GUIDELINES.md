# Bounty Guidelines

## Overview
This document outlines the process for submitting, validating, and receiving payouts for bounties in our ecosystem.

## Bounty Submission Process

### 1. Finding Available Bounties
- Check the [Issues](../../issues) tab for bounties labeled with `bounty`
- Look for the bounty amount in the issue title (e.g., `[EASY BOUNTY: 5 RTC]`)
- Read the full requirements and acceptance criteria

### 2. Claiming a Bounty
- Comment on the bounty issue with "I'd like to work on this"
- Wait for confirmation from maintainers before starting work
- Only one contributor per bounty unless specified otherwise

### 3. Submission Requirements
- Fork the repository and create a new branch
- Complete all requirements listed in the bounty issue
- Submit a pull request (PR) with:
  - Clear title referencing the bounty issue
  - Description of changes made
  - Screenshots or evidence where applicable
  - Link to the bounty issue in PR description

## Validation Criteria

### Quality Standards
- All submissions must meet the project's coding standards
- Documentation must be clear and complete
- Changes must not break existing functionality
- Follow the existing code style and conventions

### Review Process
1. **Initial Review** (24-48 hours): Maintainers check basic requirements
2. **Technical Review** (2-5 days): Detailed code/content review
3. **Testing** (1-3 days): Functionality and integration testing
4. **Final Approval**: Maintainer approval and merge

### Acceptance Criteria
- All requirements from the bounty issue are fulfilled
- Code passes all existing tests
- New features include appropriate tests
- Documentation is updated as needed
- PR passes all automated checks

## Payout Procedures

### Payment Methods
- RTC (ReturnCoin) - Primary payment method
- Alternative cryptocurrencies (case-by-case basis)
- PayPal or bank transfer (for significant bounties, KYC required)

### Payout Timeline
- **Easy Bounties (≤10 RTC)**: 1-3 business days after merge
- **Medium Bounties (11-50 RTC)**: 3-7 business days after merge
- **Hard Bounties (>50 RTC)**: 5-14 business days after merge

### Payout Process
1. PR is merged by maintainer
2. Contributor provides payment address/details
3. Payment is processed by treasury manager
4. Confirmation sent to contributor

## Bounty Categories

### Easy Bounties (1-10 RTC)
- Documentation updates
- Simple bug fixes
- Adding entries to lists
- Basic feature additions

### Medium Bounties (11-50 RTC)
- Complex feature development
- Integration work
- Performance improvements
- Comprehensive testing

### Hard Bounties (51+ RTC)
- Major feature development
- Architecture changes
- Security implementations
- Large-scale refactoring

## Rules and Guidelines

### General Rules
- One bounty per contributor at a time (unless pre-approved)
- Work must be original and not plagiarized
- Contributors must have rights to submit the work
- Maintainers reserve the right to reject submissions

### Communication
- Use GitHub comments for bounty-related discussions
- Be respectful and professional in all interactions
- Provide regular updates if work takes longer than expected

### Disputes
- Contact maintainers directly for dispute resolution
- All decisions by maintainers are final
- Appeal process available for bounties >50 RTC

## Getting Started
1. Read the project's README and contributing guidelines
2. Browse available bounties in the Issues tab
3. Start with easy bounties to familiarize yourself with the process
4. Join our community discussions for help and support
