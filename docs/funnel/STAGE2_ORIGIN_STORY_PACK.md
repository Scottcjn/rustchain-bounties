## Deliverable 2: Long-Form Article Draft

## I Refused to Throw Away My Old Macs - So I Built a Blockchain That Pays Them Instead

Every few years, the same ritual repeats itself.

Someone looks at an older computer and says some version of the same thing: it is done, it is too slow, nobody uses that anymore, it is time to recycle it and move on. The language sounds practical. Responsible, even. But underneath it is an assumption that has shaped the whole consumer technology industry: if a machine is not current, it is finished.

I have never been fully convinced by that logic.

Old computers are slower than new ones, obviously. Some cannot run the newest software. Some need patience. But many of them still boot, still compute, still connect, still do exactly what computers were built to do. What they lose in market value, they do not automatically lose in usefulness. The gap between those two ideas is where RustChain started.

The deeper frustration was not only about old Macs. It was also about what cryptocurrency mining had become. Most systems reward concentration: more capital, more specialized hardware, more energy, more scale. The message is familiar. If you want to matter, buy something newer. If you cannot keep up, you are out.

That is a strange way to design a technology that claims to be decentralized.

So the question changed. Instead of asking how to make mining more competitive for modern hardware, I started asking what would happen if a blockchain did the opposite. What if the network treated antiquity as value? What if preservation counted for something? What if the old machine in the closet had a better reason to stay powered on than sentiment alone?

That became the foundation for RustChain.

RustChain is built around a simple proposition: older, real hardware should not be pushed aside by default. In the RustChain model, hardware antiquity affects mining weight. A PowerPC G4, for example, carries a documented `2.5x` multiplier. Modern x86 hardware sits at the baseline. The point is not nostalgia for its own sake. The point is to reverse the usual incentive and make room for machines that modern computing culture has already written off.

That single design choice changes the emotional tone of the system.

Most mining stories are about maximizing output. RustChain is about recovering overlooked value. The network is not asking, "What is the fastest machine money can buy?" It is asking, "What useful machines are still here, still real, still capable of participating if we stop designing systems against them?"

That distinction matters because so much e-waste begins with a story people repeat before they test anything: old equals useless.

RustChain argues otherwise in practical terms. The node exposes a public epoch pot of `1.5 RTC`, with epochs running roughly every 10 minutes. The economics are modest and visible. They do not need hype to be interesting. The interesting part is that an older machine can still attest as real hardware and earn participation in a live network. The project's maintainers have publicly described the network as having roughly `12` active miners in this stage, which makes the system feel less like an abstraction and more like a direct invitation: if you bring a real machine online, it matters.

That is also why credibility matters so much around a project like this. If you invent giant user counts or fake payout stories, you miss the whole point. RustChain does not need exaggerated numbers to justify itself. The core idea is already strong: useful hardware should get a second life, and a network can be designed to reward that.

The onboarding path reflects that same simplicity. The miner can be installed with `pip install clawrtc`, and the project documentation centers on getting real hardware onto the network rather than selling a fantasy of effortless wealth. That is the right frame. RustChain makes more sense as proof of possibility than as a get-rich promise. If an older machine can still attest, still enroll, and still contribute, that alone is a meaningful rebuttal to planned obsolescence.

And that is the larger reason the project exists.

Technology culture often treats replacement as progress. A machine loses official support, falls behind on benchmarks, or starts missing the newest features, and suddenly the default answer is disposal. But there is another path: preserve what still works, give it work that suits it, and build systems that respect longevity instead of punishing it.

RustChain is that argument, turned into infrastructure.

It says that value does not only live in the newest devices. It says old hardware can still be part of the future if we are willing to design for it. It says the machine you were told to throw away may still have enough life left to participate in a network built with different priorities.

That is why the old Macs matter. Not because they are trendy. Not because they will beat modern systems on raw performance. But because they still work, and because building with that fact in mind creates a more interesting kind of network than one built on endless replacement.

If you have an older machine sitting idle, that is the invitation. Do not start by assuming it is done. Start by testing the assumption. Visit `rustchain.org`, install the miner, run it on real hardware, and see whether the computer you were told to discard still has something left to contribute.

Sometimes the most radical technical decision is not building for the newest machine in the room.

Sometimes it is refusing to throw away the ones that are still here.
