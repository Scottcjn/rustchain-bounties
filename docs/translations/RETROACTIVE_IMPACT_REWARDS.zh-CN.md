# 追溯性影响奖励 (Retroactive Impact Rewards)

**“达成‘什么是有用的’共识，比达成‘什么将是有用的’共识更容易。”** — Vitalik Buterin

### 运作方式

每个月，维护者会审查过去 30 天内的所有贡献，并将 **奖金池** 分配给那些产生最大实际影响的工作。这不会提前宣布——你无法通过投机取巧来获得。

### 每月奖金池：100 RTC

分配给当月排名前 3-5 位最有影响力的贡献，无论它们是否与具体的赏金任务挂钩。

### 什么是“影响”

- 修复了实际生产环境 Bug 的 PR
- 我们看到新用户实际参考的文档
- 阻止了实际攻击的安全报告
- 社区帮助——回答问题、指导新人
- 带来了可衡量的活跃新用户的内容
- 任何我们最初没考虑到设立赏金，但事实证明非常重要的工作

### 哪些不计入“影响”

- PR 的数量（10 个拼写修复 ≠ 影响）
- 自我推销
- 已经通过赏金获得全额支付的工作（不可重复领取）

### 为什么这很重要

赏金奖励我们“计划中”的工作。追溯性奖励表彰我们“发现”的有价值的工作。最好的贡献往往是那些没有人要求去做的。

该计划灵感来自 [Optimism 的 RetroPGF](https://app.optimism.io/retropgf)（该计划已向具有影响力的公共产品分发了超过 1 亿美元的奖励）。
