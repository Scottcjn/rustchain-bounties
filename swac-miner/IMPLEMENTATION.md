## 技术挑战

### 1. 内存限制 (256 字)

SWAC 仅有 256 字 × 37 位 ≈ 1.2KB 内存，而标准 SHA256 实现需要：
- 64 个 32 位 W 表 = 256 字节
- 8 个 32 位哈希状态 = 32 字节
- 8 个 32 位工作变量 = 32 字节
- 64 个 32 位 K 常量 = 256 字节
- **总计**: ~576 字节 (仅数据)

**解决方案**: 
- K 常量分页加载 (每次 8 个)
- W 表原地扩展
- 工作变量复用哈希状态区

### 2. 指令集限制

SWAC 原始指令集非常有限：
- ADD, SUB, MUL, DIV
- LD (加载), ST (存储)
- JMP (跳转), JZ (零跳转)

缺少：
- 位运算 (AND, OR, XOR, NOT)
- 移位操作 (LSL, LSR, ROR)
- 间接寻址

**解决方案**:
- 位运算通过加减法模拟 (效率低)
- 移位通过重复加减实现
- 使用自修改代码实现间接寻址

### 3. SHA256 位运算

SHA256 需要大量位运算：
```
Σ0(a) = ROTR(a,2) ⊕ ROTR(a,13) ⊕ ROTR(a,22)
Σ1(e) = ROTR(e,6) ⊕ ROTR(e,11) ⊕ ROTR(e,25)
Ch(e,f,g) = (e ∧ f) ⊕ (¬e ∧ g)
Maj(a,b,c) = (a ∧ b) ⊕ (a ∧ c) ⊕ (b ∧ c)
```

**在 SWAC 上的实现策略**:
1. 预计算旋转表 (占用内存)
2. 使用 MCU 协处理器辅助
3. 混合方案：SWAC 控制流 + MCU 位运算
