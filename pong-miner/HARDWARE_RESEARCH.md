# Atari Pong 街机 (1972) 硬件架构研究

## 概述

Atari Pong 是第一款商业成功的街机视频游戏，于 1972 年 11 月 29 日发布。与后来的游戏不同，**Pong 没有使用 CPU 或 RAM**，而是完全由**离散 TTL 逻辑芯片**构建。

## 核心硬件规格

### 逻辑技术
- **技术**: Transistor-Transistor Logic (TTL)
- **芯片系列**: 7400 系列 (Texas Instruments SN74xx)
- **工作电压**: 5V DC
- **逻辑电平**: 
  - Low: 0V - 0.8V
  - High: 2V - 5V

### 主要芯片组成

根据历史资料，Pong 街机使用了以下类型的 TTL 芯片：

| 芯片型号 | 功能 | 数量 (估计) |
|---------|------|------------|
| 7400 | 四 2 输入 NAND 门 | ~10 |
| 7404 | 六反相器 | ~5 |
| 7474 | 双 D 触发器 | ~8 |
| 7493 | 4 位二进制计数器 | ~6 |
| 74153 | 双 4 选 1 数据选择器 | ~4 |
| 7485 | 4 位幅度比较器 | ~4 |
| 74283 | 4 位二进制全加器 | ~2 |
| 74121 | 单稳态多谐振荡器 | ~2 |

**总芯片数**: 约 40-50 个 TTL 芯片

## 系统架构

### 视频信号生成

Pong 使用**模拟视频信号**直接驱动 CRT 显示器：

```
+------------------+
|  同步信号发生器   |
|  (Sync Generator)|
+--------+---------+
         |
    +----v----+
    |  视频   |
    |  逻辑   |
    +----+----+
         |
    +----v----+
    |  混合   |
    |  输出   |
    +----+----+
         |
    +----v----+
    |  CRT    |
    |  显示器  |
    +---------+
```

### 游戏逻辑模块

1. **球拍控制**
   - 使用电位器读取玩家输入
   - 垂直位置计数器 (7493)
   - 边界检测逻辑

2. **球运动逻辑**
   - X/Y 位置计数器
   - 碰撞检测 (7485 比较器)
   - 角度计算 (基于球拍击中位置)

3. **得分系统**
   - BCD 计数器 (7490)
   - 7 段显示解码器 (7447)

4. **声音生成**
   - 从同步发生器提取音调
   - 简单的方波发生器

### 内存 (无 RAM!)

Pong **没有 RAM**。所有"状态"都存储在：
- **触发器** (7474, 7473): 存储二进制状态
- **计数器** (7493, 7490): 存储位置和分数
- **移位寄存器**: 用于球轨迹

## 工作原理

### 球拍控制

```
电位器 → 比较器 → 上/下计数器 → 球拍位置
```

玩家旋转电位器改变电压，比较器决定计数器是增加还是减少。

### 球运动

```
时钟脉冲 → X/Y 计数器 → 位置比较 → 碰撞检测
                              ↓
                         方向反转
```

球的位置由两个计数器 (X 和 Y) 跟踪。当计数器值与球拍位置匹配时，触发碰撞检测。

### 碰撞检测逻辑

```
IF (ball_X == paddle_X) AND (ball_Y within paddle_range) THEN
    reverse_ball_X_direction
    increase_ball_speed (optional)
END IF
```

使用 7485 幅度比较器实现。

## 时钟系统

- **主时钟**: 约 5-10 MHz
- **水平同步**: 15.734 kHz (NTSC 标准)
- **垂直同步**: 60 Hz (NTSC 标准)

时钟信号由晶体振荡器和 74121 单稳态多谐振荡器生成。

## 功耗

- **总功耗**: 约 50-100W
- **芯片功耗**: 每个 TTL 芯片约 10-50mW (静态)
- **散热**: 需要风扇冷却

## 与现代系统的对比

| 特性 | Pong (1972) | 现代系统 |
|------|-------------|----------|
| CPU | 无 | 多核 GHz 处理器 |
| RAM | 无 | GB 级 DDR |
| 逻辑实现 | 离散 TTL 芯片 | FPGA/ASIC |
| 晶体管数 | ~500-1000 | 数十亿 |
| 功耗 | 50-100W | 10-300W |
| 成本 (当时) | ~$500 | ~$0.01 (等效功能) |

## 移植挑战

将 RustChain 矿工移植到 Pong 硬件面临以下挑战：

1. **无存储**: 无法存储复杂的矿工状态
2. **无程序**: 逻辑是硬连线的，无法"运行"代码
3. **有限状态**: 只能存储极少状态 (触发器/计数器)
4. **无计算**: 只能做简单的加减和比较

## Badge Only 方案

鉴于上述限制，我们提出**Badge Only**移植方案：

- **概念**: 创建一个象征性的"矿工徽章"，在 Pong 硬件上显示
- **实现**: 使用球的位置或得分显示来代表"挖矿"状态
- **文档**: 详细记录这个概念验证作为历史致敬

详见 `BADGE_ONLY_DESIGN.md`
