# Badge Only 移植方案设计

## 概述

由于 Atari Pong (1972) 使用**纯 TTL 逻辑**，没有 CPU、没有 RAM、无法运行程序，我们提出**Badge Only**方案作为概念验证。

## 设计理念

> "在人类历史上第一款成功街机游戏上，留下 RustChain 矿工的永恒印记"

## 方案细节

### 1. 视觉徽章 (Visual Badge)

利用 Pong 现有的显示元素来创建"矿工徽章"：

#### 方案 A: 得分显示
```
玩家 1 得分: R (RustChain 标识)
玩家 2 得分: T (Token 标识)
```

**实现**:
- 修改 7 段显示解码器逻辑
- 在特定游戏状态下显示"R"和"T"
- 需要额外的 ROM 或二极管矩阵

#### 方案 B: 球轨迹徽章
```
当球到达特定位置时，形成 RustChain logo 的简化图案
```

**实现**:
- 修改球的碰撞逻辑
- 在特定位置"反弹"形成图案
- 使用额外的计数器控制

#### 方案 C: 球拍形状修改
```
将球拍显示从直线改为简化的"RC"形状
```

**实现**:
- 修改视频生成逻辑
- 需要额外的字符发生器 ROM

### 2. 挖矿状态指示

使用游戏状态来象征"挖矿"：

| 游戏状态 | 挖矿含义 |
|---------|---------|
| 球在运动 | 正在挖矿 |
| 得分增加 | 找到区块 |
| 球速加快 | 难度增加 |
| 比赛结束 | 完成一轮挖矿 |

### 3. 硬件修改清单

#### 必要修改:
1. **7 段显示修改**
   - 添加自定义字符"R"和"C"
   - 需要修改 7447 解码器输出
   - 添加二极管矩阵

2. **状态指示器**
   - 添加 LED 指示"挖矿中"
   - 连接到球运动信号

3. **钱包地址铭牌**
   - 物理铭牌贴在机柜上
   - 激光雕刻钱包地址:
   ```
   RTC4325af95d26d59c3ef025963656d22af638bb96b
   ```

#### 可选修改:
1. **声音修改**
   - 修改音调表示"挖矿成功"
   - 使用不同的频率

2. **控制修改**
   - 添加"开始挖矿"按钮
   - 触发自动演示模式

## 简化实现方案

考虑到实际硬件修改的复杂性，我们提供**软件模拟**版本：

### Python 模拟器

创建一个 Pong 模拟器，在其中实现 RustChain 矿工徽章：

```python
# pong_miner_simulator.py
class PongMiner:
    - 显示 RustChain logo
    - 模拟"挖矿"过程
    - 显示钱包地址
    - 记录"挖矿"统计
```
